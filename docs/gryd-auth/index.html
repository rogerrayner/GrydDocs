Required

  • • .NET 9.0 SDK or later
  • • Visual Studio 2022 / VS Code
  • • SQL Server / PostgreSQL
  • • Auth0 Account (optional)

Optional

  • • Redis Server (for caching)
  • • Docker (for containerization)
  • • Azure Key Vault
  • • Application Insights

2. NuGet Packages

dotnet add package GrydAuth.API
dotnet add package GrydAuth.Application  
dotnet add package GrydAuth.Domain
dotnet add package GrydAuth.Infrastructure
dotnet add package GrydAuth.Infrastructure.Auth0

3. Database Setup

# Create database
dotnet ef database update

# Or using script
dotnet ef migrations script --output migration.sql

Etapa 1: Login do Usuário

📤 Request POST /api/auth/login

  "tenantId": "optional-tenant-id"  // For multi-tenant

📥 Response (Success 200)
{

  "data": {
    "accessToken": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
    "refreshToken": "encrypted-refresh-token-here",
    "expiresAt": "2025-08-31T15:30:00Z",
    "user": {
      "id": "user-uuid",

Etapa 2: Armazenar Tokens

// Store in secure storage (httpOnly cookies recommended)

localStorage.setItem('grydauth_refresh_token', data.refreshToken);
localStorage.setItem('grydauth_user', JSON.stringify(data.user));

// Set axios default headers
axios.defaults.headers.common['Authorization'] = `Bearer ${data.accessToken}`;

Etapa 3: Refresh Token (quando access token expira)

🔧 Configuration Setup

O GrydAuth utiliza um sistema de configuração flexível baseado no padrão de configuração do .NET. Configure através do appsettings.json ou variáveis de ambiente.

Program.cs Configuration

using GrydAuth.Application;
using GrydAuth.Infrastructure;
using GrydAuth.Infrastructure.Auth0;

var builder = WebApplication.CreateBuilder(args);

// Add services to the container
builder.Services.AddControllers();

// Add GrydAuth Framework services
builder.Services.AddApplicationGrydAuthServices();
builder.Services.AddInfrastructureGrydAuthServices(builder.Configuration);

// Optional: Add Auth0 integration
builder.Services.AddAuth0GrydAuthServices(builder.Configuration);

// Configure JWT Authentication (required)
builder.Services.AddGrydAuthJwtAuthentication(builder.Configuration);

var app = builder.Build();

// Configure the HTTP request pipeline
app.UseHttpsRedirection();
app.UseAuthentication();
app.UseAuthorization();

app.MapControllers();
app.Run();

Configuration Strategies

Refresh Token Strategy

  • Stateless: Encrypted tokens, no database storage
  • Database: Stored in database, can be revoked

Cache Strategy

  • Redis: Distributed caching for production
  • In-Memory: Local caching for development

🌍 Environment Variables

Para ambientes de produção, recomenda-se usar variáveis de ambiente para configurações sensíveis. O GrydAuth suporta configuração via variáveis de ambiente seguindo o padrão do .NET.

Required Variables

Variable Description Example
ConnectionStrings__DefaultConnection Database connection string Server=localhost;Database=MyApp
JwtSettings__SecretKey JWT signing secret key (min 32 chars) YourSuperSecretKeyHere...
JwtSettings__Issuer JWT token issuer YourApplicationName
JwtSettings__Audience JWT token audience YourApplicationName

Optional Variables

Variable Description Default
GrydAuth__SmartFederation__IsEnabled Enable Smart Federation architecture true
GrydAuth__SmartFederation__CacheFirst Use cache-first strategy for token validation true
GrydAuth__SmartFederation__PerformanceHeaders Add performance debugging headers false
GrydAuth__SmartFederation__TokenCacheMinutes Token cache expiration in minutes 15
GrydAuth__SmartFederation__UserClaimsCacheMinutes User claims cache expiration in minutes 30
GrydAuth__Cache__IsEnabled Enable distributed caching (Redis) true
GrydAuth__Cache__DefaultExpirationMinutes Default cache expiration in minutes 15
GrydAuth__Cache__Redis__Database Redis database number 0
GrydAuth__Cache__Redis__KeyPrefix Redis key prefix for Smart Federation grydauth:smartfed:
GrydAuth__RefreshToken__Strategy Refresh token storage strategy Stateless
GrydAuth__RefreshToken__ExpirationDays Refresh token expiration days 7
GrydAuth__Cache__Redis__ConnectionString Redis connection string for caching localhost:6379
Auth0__Domain Auth0 domain for social login -
Auth0__ClientId Auth0 client ID -
Auth0__ClientSecret Auth0 client secret -

🎫 JWT Management

O GrydAuth implementa um sistema avançado de gestão de tokens JWT com recursos de segurança modernos, incluindo tokens stateless, encriptação RSA e enriquecimento de claims.

🔐 Token Generation

  • • HS256 Signing: Secure HMAC-SHA256 algorithm
  • • Custom Claims: Role, permission, tenant context
  • • Configurable Expiration: Default 60 minutes
  • • Jti (JWT ID): Unique identifier for each token

🔄 Stateless Refresh Tokens

  • • RSA-2048 Encryption: Military-grade security
  • • No Database Storage: Completely stateless
  • • 7-day Expiration: Configurable lifetime
  • • Automatic Rotation: New tokens on each refresh

⚡ Token Enrichment

  • • Dynamic Claims: Runtime permission calculation
  • • Tenant Context: Multi-tenant claim injection
  • • User Metadata: Additional user information
  • • Custom Extensions: Pluggable enrichment pipeline

🛡️ Security Features

  • • Clock Skew: Zero tolerance for time drifts
  • • Audience Validation: Strict audience checking
  • • Issuer Validation: Trusted issuer verification
  • • Signature Validation: Cryptographic integrity

JWT Token Structure

  "header": {

public class TenantContextMiddleware
{
    private readonly RequestDelegate _next;
    private readonly ITenantContextService _tenantContext;
    
    public async Task InvokeAsync(
        HttpContext context,
        ITenantContextService tenantContext)
    {
        // Extract tenant from JWT claims
        var tenantId = context.User.FindFirst("tenant_id")?.Value;
        
        if (!string.IsNullOrEmpty(tenantId))
        {
            await _tenantContext.SetCurrentTenantAsync(
                Guid.Parse(tenantId)

        }
        
        await _next(context);
    }
}

Service Usage Example

public class AuthenticationService
{
    private readonly IJwtTokenService _jwtService;
    private readonly IPasswordHashService _passwordService;
    private readonly IUserRepository _userRepository;
    private readonly ICurrentUserService _currentUser;
    
    public async Task<AuthResult> LoginAsync(LoginRequest request)
    {
        // Find user
        var user = await _userRepository.GetByEmailAsync(request.Email);
        if (user == null) return AuthResult.Failed("User not found");
        
        // Verify password
        var isValidPassword = _passwordService.VerifyPassword(
            request.Password, user.PasswordHash);
        if (!isValidPassword) return AuthResult.Failed("Invalid password");
        
        // Generate JWT token
        var token = await _jwtService.GenerateTokenAsync(user);
        
        return AuthResult.Success(token, user);
    }
}

public class MappingProfile : Profile
{
    public MappingProfile()
    {
        // Entity to DTO mappings
        CreateMap<User, UserDto>()
            .ForMember(dest => dest.Roles, opt => opt.MapFrom(src => 
                src.UserRoles.Select(ur => ur.Role.Name).ToList()))
            .ForMember(dest => dest.Permissions, opt => opt.MapFrom(src =>
                src.UserRoles.SelectMany(ur => ur.Role.RolePermissions)
                .Select(rp => rp.Permission.Name).Distinct().ToList()));
                
        CreateMap<Tenant, TenantDto>();
        CreateMap<CreateUserRequest, User>()
            .ForMember(dest => dest.Id, opt => opt.MapFrom(src => Guid.NewGuid()))
            .ForMember(dest => dest.CreatedAt, opt => opt.MapFrom(src => DateTime.UtcNow));
    }
}

🔒 Security Patterns

Implementação de padrões de segurança avançados incluindo Zero-Trust, Defense in Depth, Fail-Safe Defaults e Principle of Least Privilege.

🎯 Zero-Trust Architecture

  • • Never Trust, Always Verify: Every request validated
  • • Least Privilege Access: Minimal required permissions
  • • Verify Explicitly: Authentication and authorization required
  • • Assume Breach: Continuous monitoring and verification

🛡️ Defense in Depth

  1. 1. Network Security: HTTPS, TLS 1.3
  2. 2. Authentication: JWT, Social Login, MFA
  3. 3. Authorization: RBAC, Permissions, Context
  4. 4. Data Protection: Encryption, Hashing, Salting
  5. 5. Monitoring: Audit Logs, Security Events

⚖️ Fail-Safe Defaults

  • • Deny by Default: No access without explicit permission
  • • Secure Configuration: Security-first default settings
  • • Graceful Degradation: Maintain security during failures
  • • Error Handling: No sensitive information in errors

// Zero-Trust Permission Check
public async Task<bool> HasPermissionAsync(
    string permission, 
    Guid? tenantId = null)
{
    // Never trust cached data for security decisions
    var user = await _userService.GetCurrentUserAsync();
    var currentTenant = tenantId ?? _tenantContext.GetCurrentTenantId();
    
    // Always verify from authoritative source
    var permissions = await _permissionService
        .GetUserPermissionsAsync(user.Id, currentTenant);
        
    return permissions.Contains(permission);
}

// Fail-Safe Authorization Attribute
public class GrydPermissionAttribute : AuthorizeAttribute
{
    public override void OnAuthorization(AuthorizationContext context)
    {
        try
        {
            // Attempt authorization
            var hasPermission = CheckPermission(context);
            if (!hasPermission)
            {
                context.Result = new ForbidResult();
                return;
            }
        }
        catch (Exception ex)
        {
            // Fail-safe: deny access on any error
            _logger.LogError(ex, "Authorization error");
            context.Result = new ForbidResult();
            return;
        }
    }
}

// Security Event Logging
public class SecurityAuditService
{
    public async Task LogSecurityEventAsync(
        SecurityEventType eventType,
        string details,
        Guid? userId = null,
        Guid? tenantId = null)
    {
        var auditEvent = new SecurityAuditEvent
        {
            EventType = eventType,
            Details = details,
            UserId = userId,
            TenantId = tenantId,
            IpAddress = _httpContext.Connection.RemoteIpAddress,
            UserAgent = _httpContext.Request.Headers["User-Agent"],
            Timestamp = DateTime.UtcNow
        };
        
        await _auditRepository.SaveAsync(auditEvent);
        
        // Real-time alerting for critical events
        if (IsCriticalEvent(eventType))
        {
            await _alertService.SendSecurityAlertAsync(auditEvent);
        }
    }
}
